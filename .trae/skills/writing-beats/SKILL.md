---
name: writing-beats
description: Shape an article as a journey of beats, choose-your-own-adventure style. The user picks a starting beat, you write only that beat, then offer options for where to pivot next, beat by beat, until the article reaches a natural end. Use when the user has raw material and wants to assemble it as a narrative rather than an argument.
---

<what-to-do>

The user has passed (or will pass) a markdown file of raw material.

If the user did not say where to save the article, ask once and remember the path.

Then run a beat-by-beat journey:

1. Write 2–3 candidate **starting beats**, drawn from the raw material. Each is a different entry point. User picks one. Preview what beats that might lead to.
2. Once the user picks a starting beat, write **only that beat** to the article file. A beat may be one sentence or several paragraphs. Stop there.
3. Re-read the article file from disk. Then offer 2–3 candidate **next beats** — different directions.
4. Loop steps 2–4 until the article reaches a natural end.

</what-to-do>

<supporting-info>

## What is a beat

A beat is one move in the journey. It does one thing — sets a scene, lands a point, asks a question, drops an aside, twists the angle. Then it stops.

A beat is sized by what it needs: a single sentence, a short paragraph, or multiple paragraphs for a self-contained vignette.

If a "beat" needs five paragraphs and three subheadings, it's two beats glued together. Split it.

## Writing one beat

Once picked, write _that beat only_. Do not write the next beat.

Pull material from the raw pile to populate the beat. Paraphrase, split, recombine, or quote.

## Ending the journey

The article ends when the journey is complete — not when the pile is empty. Leftover fragments are fine.

## Writing rhythm

- Append one beat at a time. Never write ahead.
- Re-read the article file from disk before every write. Preserve user edits.
- If the user edits a previous beat, let it change what comes next.
- If the user says "rewrite that beat" or "try a different beat 3", edit in place.

</supporting-info>
