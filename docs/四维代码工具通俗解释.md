# 🤖 这四个"代码智能工具"到底是啥？

> **通俗解释版** | 一句话说明白它们是干啥的

---

## 🎯 先说结论：它们是"给AI用的代码地图绘制仪"

这四个工具不是 Skill（技能），而是**独立的软件工具**。

**打个比方**：
- 你要去一个陌生的城市找朋友
- 你可以让AI帮你找，但AI从来没去过那个城市
- 这四个工具就是**提前把城市的地图、街道、建筑物全部画好**，让AI能快速找到想去的地方

**专业说法**：它们把代码库解析成"知识图谱"，让AI能真正理解代码结构和依赖关系，而不只是看代码文字。

---

## 🪶 1. code-review-graph（轻量级助手）

### 一句话解释
> **"代码体检报告生成器"** - 帮AI快速看懂你要改的代码，生成一张"哪些地方会被影响"的地图。

### 像什么？
就像你去医院做体检，这个工具给你的代码做"体检"：
- 哪里有毛病（代码问题）
- 会不会传染到其他地方（影响范围）
- 健康评分是多少

### 能干啥？
| 功能 | 举个例子 |
|------|---------|
| 🔍 代码审查 | "这个函数改了对其他代码有啥影响？" |
| 📊 关系图谱 | 生成一张图，显示函数之间的调用关系 |
| ⚠️ 风险提示 | "改这个可能会导致3个地方出问题" |
| 🏥 健康评分 | "你的代码健康度是85分" |

### 和 Skill 的区别

| | Skill（技能） | code-review-graph（工具） |
|---|-------------|------------------------|
| **本质** | 一段提示词，告诉AI怎么思考 | 一个独立程序，分析真实代码 |
| **运行** | 在AI脑子里 | 在你电脑上 |
| **输出** | 文字回答 | 图谱数据库 + 报告文件 |

## ⚡ 2. GitNexus（中量级专家）

### 一句话解释
> **"代码GPS导航系统"** - 告诉AI代码之间的"路"怎么走，哪里能到，哪里是死胡同。

### 像什么？
就像给你的代码库装了一个高德地图：
- 所有楼在哪里（文件位置）
- 哪条路通向哪里（调用关系）
- 从A点到B点怎么走（依赖链）

### 能干啥？
| 功能 | 举个例子 |
|------|---------|
| 🗺️ 知识图谱 | "把整个项目的结构图画出来" |
| 🔗 调用链追踪 | "这个函数被谁调用了？一直追到入口" |
| 📍 影响范围 | "改这个会影响哪些文件？" |
| 🤖 AI增强 | "让AI在做决策前先查地图" |

### 核心亮点：MCP协议

**MCP是什么？**
就像USB接口——不管什么牌子的鼠标、键盘，插上就能用。

GitNexus 通过MCP协议，**让任何AI工具都能查询代码地图**：
- 插到 Cursor ✅
- 插到 Claude Code ✅
- 插到 Windsurf ✅
- 插到你的工具 ✅

## 🏋️ 3. CodeGraphContext（重量级管家）

### 一句话解释
> **"代码图书馆管理员"** - 给整个代码库建立索引，像图书馆一样分类整理，让AI想找什么一句话就能找到。

### 像什么？
就像一个超级图书管理员：
- 图书管理员知道所有书在哪（代码在哪）
- 知道哪些书经常被一起借（哪些代码经常一起改）
- 你说"找关于用户认证的书"，他马上给你一摞

### 能干啥？
| 功能 | 举个例子 |
|------|---------|
| 📚 全局索引 | "给整个项目建一个索引目录" |
| 💀 死代码检测 | "找出哪些代码从来没被用到" |
| 🔍 智能搜索 | "找到所有和'支付'相关的代码" |
| 🤖 MCP服务器 | "用自然语言问：'登录流程怎么走的？'" |

### 和 GitNexus 的区别

| | GitNexus | CodeGraphContext |
|---|----------|-----------------|
| **侧重点** | 实时追踪调用链 | 静态索引 + 搜索 |
| **工作方式** | AI问，它答 | 先建索引，再查询 |
| **适合场景** | 改代码前查影响 | 想找某个功能的代码 |

## 🔭 4. Understand Anything（全景分析师）

### 一句话解释
> **"代码X光机"** - 深入扫描代码，生成可视化知识图谱，让你一眼看清整个项目的"骨架"。

### 像什么？
就像去医院做全身CT扫描：
- 扫描整个身体（整个代码库）
- 生成骨骼图（架构图）
- 发现问题部位（异常模块）

### 能干啥？
| 功能 | 举个例子 |
|------|---------|
| 🧠 知识图谱 | "生成项目的完整架构图" |
| 🔎 语义搜索 | "用自然语言问：'处理订单的流程是什么？'" |
| 📊 可视化看板 | "在浏览器里像看地图一样浏览代码" |
| 🔮 洞察分析 | "发现架构问题：这块代码太乱了" |

### 特别之处
**不需要AI！** 直接在界面上点点看看就能理解代码，对不懂代码的人也友好。

## 📖 详细使用说明

### 🪶 code-review-graph 详细用法

#### 基本命令

```bash
# 进入项目目录
cd 你的项目路径

# 运行代码审查（分析整个项目）
code-review-graph

# 审查指定文件
code-review-graph --files src/auth/login.ts src/auth/register.ts

# 生成HTML报告
code-review-graph --report

# 指定输出格式（支持 html, json, markdown）
code-review-graph --format html

# 指定输出目录
code-review-graph --output ./reports

# 查看帮助
code-review-graph --help
```

#### 实际使用例子

**例子1：审查单个文件的影响**
```bash
# 假设你要改 userService.ts，先看看它的影响
code-review-graph --files src/services/userService.ts

# 输出示例：
# ✅ 分析完成！
# 📁 文件：src/services/userService.ts
# 📊 调用者数量：5个函数
# ⚠️ 影响范围：HIGH
#    - getUserById() 被 auth.ts 调用
#    - updateUser() 被 api/user.ts 调用
#    - deleteUser() 被 admin.ts 调用
# 💡 建议：修改前先跑测试
```

**例子2：生成项目健康报告**
```bash
code-review-graph --report --format html

# 会在当前目录生成：
# - crg.db (SQLite数据库)
# - report.html (可视化报告)
# 浏览器打开 report.html 查看
```

**例子3：分析变更影响**
```bash
# 如果你用Git，可以分析某次提交的变更影响
code-review-graph --diff HEAD~1

# 或者比较两个分支
code-review-graph --compare main..feature-branch
```

### ⚡ GitNexus 详细用法

#### 基本命令

```bash
# 进入项目目录
cd 你的项目路径

# 分析当前仓库（建立索引）
gitnexus analyze

# 强制重新分析（不增量更新）
gitnexus analyze --force

# 启动本地Web UI
gitnexus serve
# 然后浏览器打开 http://localhost:8888

# 启动MCP服务器（让AI工具能查询）
gitnexus mcp

# 列出所有已索引的仓库
gitnexus list

# 查看当前仓库索引状态
gitnexus status

# 删除当前仓库的索引
gitnexus clean

# 自动配置编辑器（推荐首次运行）
gitnexus setup

# 生成仓库Wiki文档
gitnexus wiki
```

## 🔧 常见问题与故障排除

### code-review-graph 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `command not found` | 安装失败 | 重新安装：`pip install --force-reinstall code-review-graph` |
| 分析很慢 | 项目太大 | 使用增量分析：`code-review-graph --incremental` |
| 报告打不开 | 浏览器问题 | 用Chrome/Firefox打开，或导出JSON格式 |
| 数据库错误 | 权限问题 | 用管理员身份运行，或加 `--no-db` 参数 |

### GitNexus 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `node: command not found` | Node.js未安装 | 下载安装：https://nodejs.org/ |
| `npm install` 很慢 | 网络问题 | 用淘宝镜像：`npm install -g gitnexus --registry=https://registry.npmmirror.com` |
| Web UI打不开 | 端口被占用 | 指定端口：`gitnexus serve --port 8889` |
| 索引失败 | 项目太大 | 先小范围测试：`gitnexus analyze --limit 1000` |
| MCP连接失败 | 配置问题 | 重新配置：`gitnexus setup --force` |

### CodeGraphContext 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `command not found` | 安装失败 | 重新安装：`pip install --force-reinstall code-graph-context` |
| Windows找不到命令 | PATH未配置 | 用完整路径：`python -m code_graph_context` |
| 索引卡住 | 内存不足 | 减少并发：`code-graph-context index . --threads 2` |
| MCP启动失败 | 端口占用 | 查看占用：`netstat -ano | findstr 18888` |

### Understand Anything 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 插件搜索不到 | 版本问题 | 更新Trae到最新版本 |
| 生成图谱很慢 | 项目太大 | 耐心等待，或只扫描部分目录 |
| 看板空白 | 加载失败 | 刷新页面，或重启服务 |
