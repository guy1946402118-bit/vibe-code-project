# 🔧 四维代码智能协同工作流指南

> **版本**: v1.0 | **更新**: 2026-05-12
> **定位**: 轻量级助手 → 中量级专家 → 重量级管家 → 全景分析师

---

## 🎯 四维体系概述

| 工具 | 定位 | 核心能力 | 适用场景 |
|------|------|---------|---------|
| **code-review-graph** | 🪶 轻量级助手 | 快速代码审查 + 关系图谱 | 日常 Code Review、PR 检查 |
| **GitNexus** | ⚡ 中量级专家 | 调用链分析 + 影响范围评估 | 改代码前的风险评估 |
| **CodeGraphContext** | 🏋️ 重量级管家 | 全局索引 + 死代码检测 + MCP 服务器 | 深度分析和自动化 |
| **Understand Anything** | 🔭 全景分析师 | 知识图谱生成 + 自然语言查询 | 架构探索和文档生成 |

## 🚀 安装步骤

### 1️⃣ 安装 code-review-graph（轻量级助手）

```bash
# 在 Trae 终端执行
pip install code-review-graph

# 进入项目目录，运行：
code-review-graph

# 生成的文件：crg.db（SQLite 数据库）
```

**验证**：
```bash
ls crg.db  # 确认文件生成
```

---

### 2️⃣ 安装 GitNexus（中量级专家）

**前提条件**：
- Node.js 18+ （勾选 "Add to PATH"）
- Git

```bash
# 在 Trae 终端执行
npm install -g gitnexus --legacy-peer-deps

# 进入项目根目录，运行：
gitnexus

# 生成的数据：.gitnexus 文件夹
```

**启动可视化界面**：
```bash
# 在浏览器中打开
http://localhost:8888
```

**验证**：
```bash
gitnexus --version  # 确认安装成功
```

---

### 3️⃣ 安装 CodeGraphContext（重量级管家）

```bash
# 在 Trae 终端执行
pip install code-graph-context

# 若提示 "命令未找到"，运行修复脚本：
curl -o post_install_fix.sh https://raw.githubusercontent.com/code-graph-context/code-graph-context/main/scripts/post_install_fix.sh
bash post_install_fix.sh
```

**索引代码**：
```bash
# 在项目目录下运行：
code-graph-context index .

# 查询关系：
code-graph-context analyze callers my_function
code-graph-context analyze dead-code
```

**启动 MCP 服务器**（可选）：
```bash
code-graph-context mcp start

# 在 Trae 中配置智能体，即可通过自然语言查询代码关系
```

**验证**：
```bash
code-graph-context list  # 显示已索引的项目
```

## 🔄 协同工作流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    🔄 四维协同工作流                             │
│                                                                 │
│  第一阶段：探索（Understand Anything - 全景分析师）              │
│  ├── 用户提出问题："用户认证流程是怎么工作的？"                  │
│  ├── Understand 生成知识图谱，定位关键模块                      │
│  └── 输出：架构概览 + 关键文件列表                               │
│                         ↓                                       │
│  第二阶段：分析（CodeGraphContext - 重量级管家）                  │
│  ├── 索引整个代码库，建立全局关系                                │
│  ├── 检测死代码、未使用的函数                                   │
│  ├── 通过 MCP 服务器提供自然语言查询                            │
│  └── 输出：完整调用链 + 依赖关系                                │
│                         ↓                                       │
│  第三阶段：编码（GitNexus - 中量级专家）                         │
│  ├── 分析改代码的影响范围                                       │
│  ├── 评估风险等级（LOW/MEDIUM/HIGH/CRITICAL）                   │
│  ├── 列出所有直接和间接调用者                                   │
│  └── 输出：影响分析报告 + 安全建议                               │
│                         ↓                                       │
│  第四阶段：验证（code-review-graph - 轻量级助手）                │
│  ├── 快速审查代码变更                                           │
│  ├── 检查代码规范、安全漏洞                                     │
│  ├── 生成代码关系图谱                                           │
│  └── 输出：审查报告 + 改进建议                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 💡 实战示例

### 场景：修改用户认证函数

#### 第一步：用 Understand Anything 探索架构
```
用户输入：
"我想了解用户认证的整体流程"

Understand 输出：
- 关键模块：auth.ts, jwt.ts, middleware/auth.ts
- 流程：登录 → 验证 → 生成Token → 保护路由
- 知识图谱可视化
```

#### 第二步：用 CodeGraphContext 深度分析
```bash
# 查询认证相关的所有调用
code-graph-context analyze callers verifyToken

# 检测死代码
code-graph-context analyze dead-code --module auth

# 启动 MCP 服务器进行自然语言查询
code-graph-context mcp start
```

#### 第三步：用 GitNexus 评估影响
```bash
# 分析改 verifyToken 的影响
gitnexus impact --function verifyToken

# 输出：
# - 直接调用者：12个函数
# - 间接调用者：34个函数
# - 风险等级：HIGH
# - 建议：先添加测试用例
```

#### 第四步：用 code-review-graph 最终验证
```bash
# 对变更进行快速审查
code-review-graph --files auth.ts jwt.ts

# 输出：
# - 发现 2 个潜在安全问题
# - 建议优化 3 处代码
# - 生成代码关系图谱 crg.db
```

## 📋 命令速查表

### code-review-graph（轻量级助手）
```bash
code-review-graph                    # 运行审查
code-review-graph --files <files>   # 审查指定文件
code-review-graph --report          # 生成报告
```

### GitNexus（中量级专家）
```bash
gitnexus                            # 启动
gitnexus analyze                    # 分析代码库
gitnexus impact --function <name>   # 影响分析
gitnexus explore                    # 探索模式
```

### CodeGraphContext（重量级管家）
```bash
code-graph-context index .          # 索引当前项目
code-graph-context list             # 列出已索引项目
code-graph-context analyze callers <func>  # 分析调用者
code-graph-context analyze dead-code      # 死代码检测
code-graph-context mcp start        # 启动 MCP 服务器
```

### Understand Anything（全景分析师）
```bash
Understand: Generate Knowledge Graph  # 右键菜单生成
understand-dashboard                  # 启动看板
```

## ⚠️ 注意事项

### 环境要求
| 工具 | Node.js | Python | 其他 |
|------|---------|--------|------|
| code-review-graph | ❌ | ✅ 3.8+ | - |
| GitNexus | ✅ 18+ | ❌ | Git |
| CodeGraphContext | ❌ | ✅ 3.8+ | - |
| Understand | ✅ 推荐 | ❌ | - |

### 网络要求
- **国内网络**：部分工具可能需要科学上网
- **GitNexus**：需要访问 GitHub
- **Understand**：可能需要访问插件市场

### 数据存储
| 工具 | 数据位置 | 数据库 |
|------|---------|--------|
| code-review-graph | 项目目录 | crg.db |
| GitNexus | 项目目录 | .gitnexus/ |
| CodeGraphContext | 本地 | SQLite/Neo4j |
| Understand | 项目目录 | .understand/ |

### 版本兼容性
- 若遇到依赖冲突，尝试升级/降级对应工具版本
- 推荐使用虚拟环境隔离依赖

## 🎓 进阶使用

### MCP 协议集成

CodeGraphContext 支持 MCP 服务器，可与其他工具集成：

```bash
# 启动 MCP 服务器
code-graph-context mcp start

# 在 Trae 中配置
# 即可通过自然语言查询代码关系
```

### 自动化工作流

创建自定义脚本串联四个工具：

```bash
#!/bin/bash
# run-full-analysis.sh

echo "=== 1. Understand: 生成知识图谱 ==="
understand-generate

echo "=== 2. CodeGraphContext: 索引代码 ==="
code-graph-context index .

echo "=== 3. GitNexus: 分析影响 ==="
gitnexus analyze

echo "=== 4. code-review-graph: 最终审查 ==="
code-review-graph --report

echo "=== 完成 ==="
```
