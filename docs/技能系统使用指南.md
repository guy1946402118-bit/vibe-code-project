# GrowthDashboard 技能系统使用指南

> **核心原则：你不是运行脚本，而是描述需求。AI 会自动匹配最合适的技能链。**

***

## 一、什么是技能系统？

技能是预先定义好的「专业能力包」，每个技能包含系统提示词、工具配置和领域知识。当你描述需求时，AI 会自动判断需要激活哪些技能，并把它们串联成一条工作流。

```
你说的              AI 自动激活的技能链
────────────────────────────────────────────────
"帮我修复登录BUG"   → diagnose → gitnexus-debugging → tdd
"帮我写个PRD"       → grill-me → to-prd → grill-with-docs → to-issues
"分析代码架构"      → zoom-out → gitnexus-exploring → understand-explain
"前端页面不好看"    → frontend-design → karpathy-guidelines
```

## 四、8 种快速任务（直接说需求即可）

以下是最常用的 8 种任务场景。你**不需要运行任何脚本**，只需要在对话中描述需求，AI 就会自动匹配技能链。

### 0. 分析代码

```
触发词：「帮我分析一下 XXX 是怎么工作的」
自动激活：zoom-out → gitnexus-exploring → understand-explain
```

### 1. 修复 Bug

```
触发词：「帮我修复这个 Bug：[描述问题]」
自动激活：karpathy-guidelines → gitnexus-debugging → diagnose
示例：  「帮我修复这个 Bug：生命之花不显示进度」
```

### 2. 开发新功能

```
触发词：「帮我开发 [功能描述]」
自动激活：karpathy-guidelines → grill-me → tdd
示例：  「帮我开发一个用户积分排行榜功能」
```

### 3. 重构代码

```
触发词：「帮我重构 XXX：[做什么]」
自动激活：gitnexus-impact-analysis → improve-codebase-architecture → gitnexus-refactoring
示例：  「帮我重构 GoalsPage：把卡片的显示逻辑抽成独立组件」
```

### 4. 项目管理（PRD / Issue）

```
触发词：「帮我写一个 PRD：[需求描述]」
自动激活：triage → to-prd → grill-with-docs → to-issues
示例：  「帮我写一个 PRD：做一个用户成就徽章系统」
```

### 5. 写文章

```
触发词：「帮我写/编辑：[文章主题]」
自动激活：writing-fragments → writing-beats → writing-shape → edit-article
示例：  「帮我写一篇关于 React 性能优化的技术博客」
```

### 6. 渗透测试

```
触发词：「帮我做渗透测试：[目标描述]」
自动激活：pentest-methodology → pentest-agent-system → pentest-toolchain
```

### 7. 多智能体设计

```
触发词：「帮我设计多智能体系统：[需求]」
自动激活：metagpt-architecture → metagpt-role-action → metagpt-sop
```

## 七、技能是如何自动触发的？

每个技能都有一个 `SKILL.md` 文件，位于 `.trae\skills\<技能名>\` 目录下。文件中定义了：

- **触发条件**：哪些关键词或场景会激活它
- **系统提示词**：告诉 AI 在处理这类任务时应该怎么思考
- **工具配置**：告知 AI 有哪些工具可用

当你说话时：

1. AI 解析你的意图
2. 在 257 个技能中搜索匹配的描述
3. 选出最适合的技能组合
4. 按顺序激活（前一个的输出作为后一个的输入）
5. 开始执行任务

你完全不需要关心这些细节 — **只需描述你想做什么，AI 自己会决定用哪些技能。**

## 八、最佳实践

### ✅ 推荐做法

- **直接描述需求**：「帮我修复 XXX 的 Bug」而不是「运行 diagnose 技能」
- **提供上下文**：告诉 AI 问题在哪发生、什么现象、期望什么结果
- **一个任务一个重点**：不要一次塞太多不相关的需求
- **大胆提要求**：257 个技能覆盖了几乎所有领域

### ❌ 不需要做的事

- 不需要记忆技能名称 — AI 自己会匹配
- 不需要运行脚本 — 脚本只是管理工具
- 不需要知道技能链是什么 — AI 自己编排
- 不需要写「请使用 XXX 技能」— 说了也不影响

## 十、常见问题

**Q：技能已经全部安装了吗？**
A：是的。257 个技能全部就位，无需额外安装。

**Q：我需要在每次对话前选一个技能吗？**
A：不需要。直接描述需求即可，AI 自动匹配。

**Q：如果 AI 用了不对的技能怎么办？**
A：直接说「你不需要用 XXX，换个方式」，AI 会重新选择。

**Q：可以自定义新技能吗？**
A：可以。直接说「帮我创建一个技能用来处理 XXX」，AI 会用 `skill-creator` 技能帮你创建。

**Q：技能会占用很多内存吗？**
A：不会。技能文件只是 Markdown 文本，总和不到 5MB，AI 只加载匹配到的几个。
